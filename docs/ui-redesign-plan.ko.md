# Glaze UI 리디자인 계획

## 문서 목적

이 문서는 현재 `Glaze`의 실제 앱 화면을 기준으로, `LookAway`가 주는 제품 인상을 어떻게 우리 앱 UI에 번역할지 정리한 설계 문서다.

목표는 `LookAway` 홈페이지를 그대로 복제하는 것이 아니다.
목표는 아래 두 가지를 동시에 만족하는 것이다.

1. 지금의 작은 MVP 구조와 안정성을 유지한다.
2. 시각 언어와 행동 설계를 `프리미엄 Mac 웰니스 툴` 쪽으로 끌어올린다.

## 비교 기준

비교에 사용한 기준은 아래다.

- 현재 `Glaze`의 실제 메뉴바 팝오버 화면
- 현재 `Glaze`의 실제 브레이크 오버레이 화면
- `LookAway` 홈페이지에 노출된 제품 데모 화면과 시각 시스템

## 핵심 결론

현재 `Glaze`는 기능 구조는 이미 맞다.

- 메뉴바 타이머
- heads-up 상태
- 브레이크 오버레이
- skip / pause / start break now

문제는 구조가 아니라 표현 방식이다.

- 지금 `Glaze`는 `잘 만든 내부용 유틸리티` 느낌에 가깝다.
- `LookAway`는 `브랜드가 살아 있는 Mac 웰니스 제품` 느낌에 가깝다.

따라서 다음 UI 작업의 중심은 기능 추가보다 아래 4개다.

1. 시각 언어 통일
2. 정보 위계 재구성
3. 브레이크 직전과 브레이크 중의 행동 분리
4. 기본 macOS 설정 패널 같은 인상 제거

## 현재 Glaze의 강점

- 메뉴바에서 남은 시간이 바로 읽힌다.
- MVP 기준 필요한 제어가 다 있다.
- 브레이크 오버레이 구조는 이미 성립한다.
- 멀티 디스플레이에서 브레이크 화면이 각 화면을 제대로 덮는다.

## 현재 Glaze의 약점

### 1. 팝오버와 오버레이가 서로 다른 제품처럼 보인다

- 팝오버는 밝은 반투명 macOS 기본 패널 느낌이다.
- 오버레이는 어두운 보라 계열이라 감정 톤이 훨씬 강하다.
- 같은 앱 안에서 시각 언어가 끊긴다.

### 2. 팝오버가 제어판처럼 보인다

- 타이머, 설명, 액션, 설정, 종료 버튼이 한 장에 비슷한 밀도로 있다.
- 사용자가 “현재 중요한 정보”보다 “조절 가능한 옵션”을 먼저 읽게 된다.

### 3. 브레이크 화면이 아직 너무 운영 화면에 가깝다

- `Skip Break`, `Pause Timer`가 브레이크 경험과 같은 무게로 보인다.
- 휴식 유도보다 즉시 조작 유도가 더 강하다.

### 4. 카피와 레이아웃이 아직 건조하다

- `Focus session`, `Timer Settings` 같은 문구는 정확하지만 감정 톤이 약하다.
- 설명문이 잘리는 구간이 있어 마감 품질이 떨어져 보인다.

## LookAway에서 가져올 요소

### 시각 언어

- 거의 검정에 가까운 어두운 배경
- 핑크, 퍼플, 앰버 계열의 제한된 강조색
- 큰 라운드
- 은은한 글로우와 깊이감
- 실제 기능 표면을 강조하는 단순한 위계

### 행동 설계

- 브레이크 직전에는 “협상 가능한 제어”를 보여준다
- 실제 브레이크 순간에는 화면을 조용하게 만든다
- 긴 설명보다 큰 타이머와 짧은 메시지로 집중시킨다

### 제품 인상

- 단순 타이머 앱이 아니라 `Mac에 잘 붙는 건강 보조도구`처럼 느껴져야 한다
- 시스템 기본 UI를 그대로 노출하기보다, 제품의 톤이 보이는 커스텀 표면이 필요하다

## 리디자인 원칙

### 1. 마케팅 사이트를 복제하지 않는다

앱은 랜딩 페이지가 아니다.
그래서 히어로 레이아웃이나 과한 장식은 가져오지 않고, 아래만 가져온다.

- 색 체계
- 표면 처리
- 카피 톤
- 정보 밀도 조절

### 2. 네이티브 Mac 느낌은 유지한다

`LookAway`처럼 보인다고 해서 웹 같은 UI를 넣지 않는다.

- SwiftUI + AppKit 구조 유지
- 버튼, 포커스, 스테퍼, 창 동작은 macOS 사용성을 해치지 않는다
- 다만 기본 스타일을 그대로 두지 않고 커스텀 컨테이너와 색 체계로 정리한다

### 3. 브레이크 화면은 덜 말하고 더 쉬게 해야 한다

- 브레이크 화면은 `메시지 + 타이머 + 최소 제어`가 중심이어야 한다
- 제어는 남기되, 시각적 우선순위는 낮춘다

### 4. 설정은 1차 경험보다 뒤로 밀어야 한다

- 사용자가 메뉴를 열었을 때 가장 먼저 읽어야 하는 것은 `현재 상태`다
- 설정은 같은 화면에 있어도 보조 영역처럼 느껴져야 한다

## 화면별 계획

## 1. 메뉴바 팝오버

### 목표

현재의 설정 패널 느낌을 줄이고, `현재 세션 상태를 보여주는 작은 대시보드`처럼 바꾼다.

### 바꿀 것

- 전체 패널을 다크 테마로 전환
- 상단에 큰 상태 카드 배치
- 타이머와 상태 문구를 더 크게
- 액션 버튼을 상태 카드 바로 아래에 배치
- 설정 영역은 별도 섹션처럼 시각 무게를 낮춤
- 하단 `Reset Cycle / Quit`는 더 가볍게 처리

### 레이아웃 초안

1. 앱 이름 + 현재 phase
2. 큰 타이머
3. 짧은 안내 문구
4. 핵심 액션 1~2개
5. 보조 설정 섹션
6. 보조 관리 액션

### 구체적 방향

- `Focus session` 같은 제목은 더 부드럽게 다듬을 수 있다
- 설명문은 1~2줄 안에서 끊기고, 잘리지 않게 폭과 폰트 크기를 다시 잡는다
- 설정 행은 현재보다 더 조용하고 밀도 높게 배치한다
- 스테퍼는 그대로 써도 되지만, 컨테이너와 배경을 바꿔 기본 설정 앱 느낌을 줄인다

## 2. Heads-up 상태

### 목표

브레이크 직전에는 사용자가 타이머와 협상할 수 있어야 한다.

### 바꿀 것

- heads-up phase의 카피를 더 행동 중심으로 바꾼다
- `Snooze 1m`, `Start Break`, `Skip` 우선순위를 명확하게 만든다
- working phase와 breaking phase 사이에 시각적으로 더 분명한 전환을 만든다

### 방향

- heads-up은 “브레이크 준비 화면”처럼 보여야 한다
- 이 단계에서 조작을 충분히 제공하고, 브레이크 화면은 오히려 단순화한다

## 3. 브레이크 오버레이

### 목표

현재 오버레이를 `전체화면 운영 패널`이 아니라 `짧은 휴식 경험`처럼 느껴지게 만든다.

### 바꿀 것

- 배경 그라디언트 퀄리티 개선
- 중앙 콘텐츠 폭 축소
- 텍스트 위계 단순화
- 액션 버튼 축소 또는 하단 보조 배치
- 타이머를 시각 중심에 유지

### 구체적 방향

- 현재 아이콘, 메시지, 타이머 구조는 유지
- 설명 문구는 더 짧고 더 차분하게
- `Skip Break`는 secondary 스타일
- `Pause Timer`는 지금보다 덜 튀게 조정
- 버튼 간격과 타이포 비율을 다시 잡아 “대시보드” 느낌을 줄인다

## 4. 공통 시각 시스템

### 목표

팝오버와 오버레이가 같은 앱처럼 보이게 만든다.

### 필요한 토큰

- 배경색
- 카드 배경색
- 강조색
- 보조 텍스트색
- 라운드 값
- 그림자/글로우 강도
- 기본 여백 단위

### 구현 방향

UI 토큰은 별도 파일로 분리한다.

예상 파일:

- `Glaze/GlazeTheme.swift`
- 또는 `Glaze/DesignTokens.swift`

이 파일에서 색/간격/코너 반경을 관리하고 `MenuBarView.swift`, `BreakOverlayManager.swift`에서 공통 사용한다.

## 구현 순서

### 1단계: 시각 토큰 도입

대상 파일:

- `Glaze/MenuBarView.swift`
- `Glaze/BreakOverlayManager.swift`
- 신규 `GlazeTheme` 파일

목표:

- 색, 라운드, 여백 체계 통일
- 팝오버와 오버레이가 같은 제품처럼 보이게 만들기

### 2단계: 메뉴바 팝오버 재구성

대상 파일:

- `Glaze/MenuBarView.swift`
- 필요 시 `Glaze/GlazeController.swift`

목표:

- 상태 카드 중심 구조로 변경
- 텍스트 잘림 제거
- 설정 섹션의 시각 무게 축소

### 3단계: 브레이크 오버레이 재구성

대상 파일:

- `Glaze/BreakOverlayManager.swift`
- 필요 시 오버레이 전용 view 분리

목표:

- 중앙 집중형 레이아웃
- 더 조용한 액션 배치
- 멀티 디스플레이 동작 유지

### 4단계: 카피 조정

대상 파일:

- `Glaze/GlazeController.swift`

목표:

- phase별 문구를 제품 톤에 맞게 다듬기
- heads-up과 breaking의 감정 차이를 더 명확하게 만들기

### 5단계: 미세 polish

목표:

- 등장/사라짐 애니메이션
- 버튼 강조 강도 조정
- spacing 미세 보정

이 단계는 안정성 검증 뒤에만 진행한다.

## 이번 리디자인에서 하지 않을 것

- 라이선스/결제 UI
- 별도 설정 윈도우
- 홈페이지 제작
- posture / blink 기능 추가
- automation 기능 추가

이번 작업은 어디까지나 `현재 MVP의 시각적/행동적 완성도 향상`에 집중한다.

## 수용 기준

리디자인 완료 판단 기준은 아래다.

- 팝오버와 오버레이가 같은 앱처럼 보인다
- 메뉴를 열었을 때 현재 상태가 가장 먼저 읽힌다
- 설명문 잘림이 없다
- 브레이크 화면에서 타이머와 메시지가 제어보다 먼저 읽힌다
- 내장/외장 모니터에서 기존처럼 안정적으로 동작한다
- 메뉴바 타이머 동작, pause/resume, skip, start break now가 깨지지 않는다

## 다음 작업 제안

이 문서를 기준으로 다음 실제 구현은 아래 순서가 적절하다.

1. 공통 테마 토큰 추가
2. 메뉴바 팝오버 재디자인
3. 브레이크 오버레이 재디자인
4. heads-up 카피와 액션 위계 정리
