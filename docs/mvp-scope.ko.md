# Glaze MVP 범위

## 목표

LookAway 류의 앱을 바로 완제품처럼 따라가는 대신, 개발하면서 실제로 켜둘 수 있는 최소 루프를 먼저 만든다.

핵심 루프는 아래다.

1. 메뉴바에서 남은 시간을 본다.
2. 작업 시간이 끝나기 직전에 heads-up 상태가 된다.
3. 시간이 다 되면 전체 화면 브레이크 오버레이가 뜬다.
4. 잠깐 쉬고 다시 작업 루프로 돌아간다.

## MVP 포함

- 메뉴바 앱
- 작업 타이머 자동 시작
- heads-up 상태
- 멀티 디스플레이 전체화면 브레이크 오버레이
- `Pause / Resume`
- `Snooze 1m`
- `Start Break Now`
- `Skip Break`
- 작업/휴식/heads-up 시간 조절
- 로컬 `UserDefaults` 저장
- 저장된 설정값 range 보정

## MVP 제외

- 회의, 통화, 영상 재생, 게임 자동 pause
- 메뉴바 외 알림 배너
- 포인터를 따라다니는 countdown
- posture / blink reminder
- 배경, 사운드, 메시지 커스터마이징
- 라이선스/결제
- 업데이트 프레임워크
- 접근성 권한 기반 고급 제어

## 구현 원칙

- `Glacier`처럼 작은 코드베이스를 유지한다.
- 의존성을 넣지 않는다.
- AppKit은 메뉴바와 오버레이 윈도우에만 쓴다.
- 상태 전이는 별도 스케줄러 객체로 고립한다.
- 지금 단계에서는 “확장성”보다 “항상 켜둘 수 있는 안정적인 루프”를 우선한다.

## 현재 검증된 것

- 메뉴바 팝오버를 연 상태에서도 타이머가 계속 간다.
- 내장 디스플레이 + 외장 모니터 조합에서 브레이크 오버레이가 각 화면을 정상적으로 덮는다.
- 브레이크 종료 후 다음 작업 루프로 자동 복귀한다.
- 비정상적인 persisted 설정값이 있어도 앱 시작 시 안전한 범위로 되돌린다.
- 작업/heads-up 중 5분 이상 idle이면 자동 pause 되고, 돌아오면 자동 resume 된다.
- 메뉴바 팝오버에서 오늘 누적 브레이크 시간과 오늘 기준 비율을 볼 수 있다.

## 현재 남은 제약

- 회의/미디어 재생 중 auto-pause는 아직 없다.
- 로그인 시 자동 실행은 아직 없다.
- 브레이크 화면 연출은 기능 위주라 LookAway 수준의 polish는 아직 아니다.
- XCTest는 스케줄러, idle auto-pause 정책, 브레이크 시간 집계 위주고, AppKit 오버레이/UI 흐름은 여전히 수동 검증에 의존한다.

## 다음 단계 후보

1. 회의/미디어 재생 감지
2. 로컬 알림 권한 요청 + heads-up notification
3. 더 부드러운 break 화면 연출
4. 로그인 시 자동 실행
5. 자세/깜빡임 reminder 분리
